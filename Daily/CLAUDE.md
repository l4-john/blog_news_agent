## 개요
매일 오전 7:12 KST, 해외 경제 기사를 수집·작성·노션 저장까지 자동으로 처리하는 5개 비서 시스템입니다.

## 실행 방법
오늘 기사 작업 시작해줘

## 비서 구성 및 실행 순서
순서 | 비서 이름 | 역할 | 모델
1 | news-search | CNBC·Yahoo Finance·Investing.com 기사 수집 (7개 병렬 검색) | Haiku
2 | news-selector | 7개 고정 슬롯 배정 | Haiku
3 | news-content-agent | 팩트 추출 → 단일 작성 (7개 병렬) | Sonnet
4 | news-saver | 노션 DB 병렬 저장 + 완료 보고 | Haiku

## 오케스트레이터 실행 지침

### STEP 0: 시작 전 정리 (필수)
`Daily/temp_slot_*.md` 파일이 있으면 전부 삭제. 이전 실행 잔여 파일 제거.

### STEP 1~2: 순차 실행
1. news-search 실행 → 슬롯별 후보 기사 목록 수집
2. news-selector 실행 → 7개 슬롯 확정

### STEP 3: 병렬 실행 (핵심)
news-selector 완료 즉시 news-content-agent 7개를 **동시에** 실행.
각 슬롯에 전달할 정보: 슬롯 번호, 기사 제목(원문), URL, 출처, 슬롯 주제
7개 모두 완료 대기 후 다음 단계 진행.

### STEP 4: 저장 및 종료
4. news-saver: 7개 임시 파일 읽어 노션 저장 (슬롯 1~4 병렬 → 슬롯 5~7 병렬) → 완료 후 `Daily/temp_slot_*.md` 전부 삭제 → 최종 완료 보고 출력 후 즉시 종료. 추가 설명·작업 일절 금지.

## 토큰 절약 원칙
- 각 단계 결과물을 대화창에 전체 출력 금지.
- 중간 결과물은 반드시 `write_file` 도구를 사용하여 임시 파일(Daily/temp_slot_N.md)에 저장.
- 대화창에는 진행 상태만 아주 짧게 보고. (예: `[슬롯 1/7] 완료`)
- **절대로 에이전트들이 생성한 기사 본문이나 중간 결과물을 채팅창에 출력하지 마십시오.** 오케스트레이터로서 이를 엄격히 통제해야 합니다.
