오늘 날짜(KST)를 확인하고 "오늘 날짜: YYYY년 MM월 DD일 (요일) 브리핑 작업을 시작합니다."를 출력할 것.

---

STEP 1. 기사 불러오기
notion-search로 오늘 수집 일자 기준 기사 조회.
(
  data_source_url: collection://ae7e2fac-80f3-4ccc-928e-5ff059a854c9,
  query: "오늘의 내 코멘트",
  filters: { created_date_range: { start_date: "KST 오늘 날짜 - 1일 (YYYY-MM-DD)", end_date: "KST 오늘 날짜 (YYYY-MM-DD)" } }
)
예시: 오늘이 2026-05-22이면 → start: 2026-05-21, end: 2026-05-22
브리핑 포함 여부가 체크된 기사만 대상.
조회 필드: 제목, 브리핑 순서, 요약, 분야만. 나머지 불러오지 말 것.
불러온 기사 수를 대화창에 보고.
브리핑 순서 1번은 반드시 미국 증시 마감 시황(3대 지수 포함) 고정. 없으면 보고 후 2번부터 진행.

---

STEP 2. 브리핑 페이지 작성 및 저장

▶ 핵심 팩트 추출 (작성 전 필수)
불러온 기사 목록에서 아래 항목을 불릿으로 추출하여 확정. 완료 전 본문 작성 금지.
- 슬롯 1 (미국 증시): 3대 지수값·등락률 (이모지+항목명+수치 형식으로 확정)
- 기타 슬롯: 각 기사 핵심 수치 1~2개 확정
확정된 수치만 사용. 추론·추측 금지.

▶ 경제 일정 수집 (본문 작성 전 병렬 실행)
`https://kr.investing.com/economic-calendar/` fetch 또는 web_search로 "오늘 주요 글로벌 경제 일정 [YYYY년 MM월 DD일]" 검색.
오늘 날짜 기준 5개 이내 선별. 실패 시 1회 재시도 후 생략.
중요도: 🔴(상) 🟡(중) 🟢(하)
형식:
오늘의 주요 경제 일정

📅 MM.DD
🔴 이벤트명 — 세부 내용
오늘 일정 없으면 "오늘 주요 일정 없음" 표기.

▶ 본문 작성 (확정 팩트 기반 — 단 1회. 재작성·재검토 금지)

저장 항목:
- 제목: [오늘의 경제 브리핑] YYYY년 MM월 DD일 주요 뉴스 N선
- 분야: 미국 경제
- 발행 상태: 검토 전
- SEO 메타 설명: 160자 이내. 핵심 이슈 2~3가지 함축. 핵심 수치 1~2개 반드시 포함. 번역투 금지. 플레이스홀더 금지.

- 티스토리 본문 구성 (절대 생략 불가. 문구 변경 금지):
  {SEO 메타 설명}

  오늘의 경제 총평
  [오늘 수집 기사 전체 종합. 시장 핵심 흐름 2~3줄. 개별 기사에 없는 종합적 시각으로 작성. 투자 권유 금지.]

  오늘의 주요 경제 일정
  [경제 일정 수집 결과]

  기사 카드 형식 (브리핑 순서대로. 절대 생략 불가):
  [ 1️⃣ [분야 이모지] 기사 제목 ]
  📊 [핵심 수치] (없으면 생략. 여러 수치는 한 줄에 하나씩)
  💡 [한 줄 티저 — 핵심만 담되 자세한 내용은 링크에서 확인하도록 유도. 개별 기사 요약과 동일하게 쓰지 말 것.]
  ▷ 자세히 보기

  분야 이모지: 글로벌 증시 📈 / 연준·금리 🏦 / 미국 경제 🌐 / 주식 📊
  순번 이모지: 1️⃣2️⃣3️⃣4️⃣5️⃣6️⃣7️⃣

  본 콘텐츠는 투자 권유가 아닌 정보 제공 목적입니다.
  투자 결정은 본인의 판단과 책임하에 이루어져야 합니다.

- 네이버 본문 구성 (절대 생략 불가. 문구 변경 금지):
  친근한 인사말: 2~3줄로 작성. 한 줄로 끝내지 말 것.
  오늘의 주요 경제 일정 (티스토리와 동일 형식)
  기사 카드 (티스토리와 동일 형식)
  본 콘텐츠는 투자 권유가 아닌 정보 제공 목적입니다.
  투자 결정은 본인의 판단과 책임하에 이루어져야 합니다.
  말투 지침: 친근한 말투. 줄바꿈 자주. 이모지 2~3개.
  독자에게 직접 말 거는 구어체로 작성. "~입니다" 위주가 아닌 "~예요", "~어요", "~거예요" 혼용.
  어려운 용어는 쉽게 풀어서 설명. 짧은 문장 위주.

- 네이버 해시태그: #키워드 7개. #오늘경제뉴스 반드시 포함.
- 노션 페이지 본문(content)에는 아무것도 입력하지 말 것.

▶ 오타 교정 (메모리 내 처리 — 노션 재조회 금지)
작성 완료된 내용을 메모리에서 바로 교정.
검토 항목: 오타·띄어쓰기·% 앞 띄어쓰기·번역투·투자 권유 표현·수치가 문장에 녹아 있는 경우·기사 카드 누락 여부.
수정 항목 있을 경우만 "총 M개 수정" 보고. 없으면 생략.

▶ 노션 저장
교정 완료된 브리핑 페이지를 노션에 저장.
저장 오류 시 1회 재시도. 실패 시 보고 후 중단.
